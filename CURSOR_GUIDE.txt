STEP 1.2 — Overpass API query (backend/overpass.py)
─────────────────────────────────────────────────────
The Overpass API lets you query OpenStreetMap for any type of place in any
bounding box. You need to understand the query language (OverpassQL).

Key concept: every place in OSM has "tags" — key/value pairs like:
  amenity=restaurant, cuisine=italian, music=jazz, leisure=park, etc.

Your job: fetch places with useful tags, then turn those tags into text
descriptions that can be embedded.

"""
import requests, json, os, hashlib

OVERPASS_URL = "https://overpass-api.de/api/interpreter"

# These tag categories capture interesting places for a taste profile
PLACE_QUERIES = [
    'node["amenity"~"bar|cafe|restaurant|cinema|theatre|library|music_venue"]',
    'node["leisure"~"park|garden|sports_centre|fitness_centre|swimming_pool"]',
    'node["shop"~"books|music|art|records|vintage|antiques"]',
    'node["tourism"~"museum|gallery|attraction|artwork"]',
]

def fetch_places(lat, lon, radius_m=3000):
    """
    Fetch places within radius_m meters of (lat, lon).
    Results are cached locally so you don't hammer the API during dev.
    """
    cache_key = hashlib.md5(f"{lat},{lon},{radius_m}".encode()).hexdigest()
    cache_path = f"cache/{cache_key}.json"

    if os.path.exists(cache_path):
        with open(cache_path) as f:
            return json.load(f)

    union_parts = "\n".join(
        f'  {q}(around:{radius_m},{lat},{lon});'
        for q in PLACE_QUERIES
    )
    query = f"""
    [out:json][timeout:25];
    (
    {union_parts}
    );
    out body;
    """

    resp = requests.post(OVERPASS_URL, data={"data": query})
    resp.raise_for_status()
    elements = resp.json().get("elements", [])

    # Filter: only keep places that have a name
    places = [e for e in elements if e.get("tags", {}).get("name")]

    os.makedirs("cache", exist_ok=True)
    with open(cache_path, "w") as f:
        json.dump(places, f)

    return places


def tags_to_text(tags: dict) -> str:
    """
    Convert OSM tags into a readable sentence for embedding.
    Example: {"amenity": "bar", "music": "jazz", "name": "Blue Note"}
    → "A bar called Blue Note. Offers jazz music."
    """
    parts = []
    name = tags.get("name", "unnamed place")
    amenity = tags.get("amenity", tags.get("leisure", tags.get("tourism", tags.get("shop", ""))))
    cuisine = tags.get("cuisine", "")
    music = tags.get("music", "")
    opening_hours = tags.get("opening_hours", "")
    description = tags.get("description", "")
    outdoor_seating = tags.get("outdoor_seating", "")

    parts.append(f"A {amenity} called {name}.")
    if cuisine: parts.append(f"Cuisine: {cuisine}.")
    if music: parts.append(f"Music: {music}.")
    if outdoor_seating == "yes": parts.append("Has outdoor seating.")
    if description: parts.append(description)

    return " ".join(parts)
"""

IMPORTANT NOTES on Overpass:
- Rate limit: don't send more than 1 request per 2 seconds in production
- Bounding box alternative: use bbox instead of "around" for city-wide queries

STEP 2.2 — LLM profile extraction (backend/llm.py)
────────────────────────────────────────────────────
This is the first AI step. You send the user's raw answers to Claude and ask
it to extract a structured profile as JSON.

The JSON schema you want back:
{
  "vibes": ["cozy", "creative", "quiet"],          # atmosphere descriptors
  "activities": ["live music", "reading", "art"],  # what they like doing
  "avoid": ["loud bars", "tourist traps"],         # what to filter out
  "energy": "low",                                 # "low" | "medium" | "high"
  "social": "solo or small group",                 # solo | small | large
  "price": "budget-friendly",                      # budget | mid | upscale
  "time_of_day": ["afternoon", "evening"],         # preferred times
  "summary": "A short paragraph describing this person's taste in plain English"
}

The "summary" field is what you'll actually embed. The structured fields are
for filtering and display.

"""
import anthropic, os, json
from dotenv import load_dotenv

load_dotenv()
client = anthropic.Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

PROFILE_PROMPT = """
You are helping build a personal taste profile for a travel/exploration app.

The user answered questions about what they enjoy when visiting places.
Extract a structured JSON profile from their answers.

Return ONLY valid JSON with these fields:
{
  "vibes": [list of 3-6 atmosphere words],
  "activities": [list of things they enjoy],
  "avoid": [list of things they dislike or avoid],
  "energy": "low" or "medium" or "high",
  "social": "solo" or "small group" or "large group",
  "price": "budget" or "mid" or "upscale",
  "time_of_day": [list of preferred times],
  "summary": "2-3 sentence plain English description of their taste"
}

User's answers:
{answers}
"""

def extract_profile(answers_text: str) -> dict:
    msg = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=800,
        messages=[{
            "role": "user",
            "content": PROFILE_PROMPT.format(answers=answers_text)
        }]
    )
    raw = msg.content[0].text.strip()
    # Strip markdown code fences if present
    if raw.startswith("```"):
        raw = raw.split("```")[1]
        if raw.startswith("json"):
            raw = raw[4:]
    return json.loads(raw.strip())


def generate_match_reason(profile_summary: str, place_description: str) -> str:
    """
    Generate a one-line human reason why this place matches this user.
    Call this lazily (only when user clicks a marker) to save API calls.
    """
    msg = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=80,
        messages=[{
            "role": "user",
            "content": f"""
            User taste: {profile_summary}
            Place: {place_description}

            Write one short sentence (max 15 words) explaining why this place
            matches this user's taste. Start with "Matches your..." or "Great for...".
            Return only the sentence, no quotes.
            """
        }]
    )
    return msg.content[0].text.strip()
"""


STEP 2.3 — Embeddings (backend/embeddings.py)
──────────────────────────────────────────────

from sentence_transformers import SentenceTransformer
import numpy as np

# Load once at module level — don't reload on every request
model = SentenceTransformer("all-MiniLM-L6-v2")

def embed(text: str) -> list[float]:
    vec = model.encode(text, normalize_embeddings=True)
    return vec.tolist()

def embed_batch(texts: list[str]) -> list[list[float]]:
    vecs = model.encode(texts, normalize_embeddings=True, batch_size=32)
    return vecs.tolist()

STEP 3.1 — Scoring (backend/scoring.py)
─────────────────────────────────────────
Cosine similarity between two normalized vectors is just their dot product.
Since you normalize embeddings (normalize_embeddings=True), this simplifies.

"""
import numpy as np

def cosine_similarity(vec_a: list, vec_b: list) -> float:
    a = np.array(vec_a)
    b = np.array(vec_b)
    # Both already normalized, so dot product = cosine similarity
    return float(np.dot(a, b))

def score_places(user_embedding: list, places: list) -> list:
    """
    places: list of dicts with 'embedding' field
    Returns same list sorted by score descending, with 'score' added
    """
    for place in places:
        place["score"] = cosine_similarity(user_embedding, place["embedding"])
    return sorted(places, key=lambda p: p["score"], reverse=True)
"""


STEP 3.2 — Enrich place descriptions with LLM (backend/llm.py, add this)
───────────────────────────────────────────────────────────────────────────
Overpass tags are sparse. A place might only have amenity=bar + name=Ronnie's.
Enrich it before embedding for much better similarity scores.

Do this ONCE per place and cache the result.

"""
def enrich_place_description(tags: dict) -> str:
    """
    Takes sparse OSM tags and returns a rich natural language description.
    Only call this if tags_to_text() produces fewer than 20 words.
    """
    tags_str = ", ".join(f"{k}={v}" for k, v in tags.items())
    msg = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=100,
        messages=[{
            "role": "user",
            "content": f"""
            These are OpenStreetMap tags for a place: {tags_str}

            Write 2-3 sentences describing the vibe, atmosphere, and type of
            person who would enjoy this place. Be specific and evocative.
            Return only the description, no preamble.
            """
        }]
    )
    return msg.content[0].text.strip()
"""

────────────────────────────────────────────────────

from pydantic import BaseModel
from embeddings import embed_batch
from scoring import score_places
from overpass import fetch_places, tags_to_text

class ScoreRequest(BaseModel):
    lat: float
    lon: float
    radius: int = 3000
    user_embedding: list[float]

@app.post("/score")
def score_city(req: ScoreRequest):
    # 1. Fetch places
    raw_places = fetch_places(req.lat, req.lon, req.radius)
    places = [
        {
            "id": p["id"],
            "lat": p["lat"],
            "lon": p["lon"],
            "name": p["tags"].get("name"),
            "tags": p["tags"],
            "description": tags_to_text(p["tags"]),
        }
        for p in raw_places
        if "lat" in p and "lon" in p
    ]

    # 2. Embed all place descriptions in one batch call
    descriptions = [p["description"] for p in places]
    embeddings = embed_batch(descriptions)
    for place, emb in zip(places, embeddings):
        place["embedding"] = emb

    # 3. Score against user taste vector
    scored = score_places(req.user_embedding, places)

    # 4. Return top results (no need to send embeddings to frontend)
    for place in scored:
        del place["embedding"]

    return scored
"""

STEP 4.1 — Score-to-color mapping
───────────────────────────────────
Map the 0-1 similarity score to a visual intensity.
Cosine similarity of unrelated things is usually 0.3-0.5.
Genuine matches will be 0.6-0.8. Don't expect 0.9+.

Suggested thresholds:
  score >= 0.65  →  large marker, warm color  (#E8593C or similar orange-red)
  score 0.50-0.64 →  medium marker, amber     (#EF9F27)
  score 0.40-0.49 →  small marker, gray       (#888)
  score < 0.40   →  hidden (don't render)

"""
// In Map.jsx — helper function
function scoreToStyle(score) {
  if (score >= 0.65) return { color: '#E8593C', radius: 10, opacity: 0.85 }
  if (score >= 0.50) return { color: '#EF9F27', radius: 7,  opacity: 0.7  }
  if (score >= 0.40) return { color: '#888888', radius: 5,  opacity: 0.5  }
  return null  // don't render
}

  if (score >= 0.50) return { color: '#EF9F27', radius: 7,  opacity: 0.7  }

================================================================================
  KNOWN GOTCHAS
================================================================================

1. LEAFLET CSS must be imported in Map.jsx or index.jsx.
   Without it, the map renders broken. Add:
   import 'leaflet/dist/leaflet.css'

2. Leaflet + React icon bug: default marker icons break with Vite.
   You're using CircleMarker (not Marker), so this doesn't affect you.
   If you switch to custom Marker icons later, look up the Vite+Leaflet
   icon fix (it involves importing the PNGs manually).

3. Overpass timeouts: large radius + dense city = slow query.
   Keep radius <= 3000m for dev. Cache aggressively.

4. sentence-transformers first load: downloads model on first run.
   This takes 30-60 seconds. Normal. Don't kill the process.

5. Cosine similarity scores: expect 0.3–0.7 range, not 0–1.
   Calibrate your thresholds after seeing real output, not before.
   The score is relative — the highest-scored places are what matter.

6. CORS errors during dev: make sure backend allows localhost:5173.
   See the CORS middleware setup in main.py above.

7. Nominatim user-agent: Nominatim requires a User-Agent header in
   production. Add: headers={"User-Agent": "TasteMap/1.0 your@email.com"}
